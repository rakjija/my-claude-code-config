# 바이브 코딩 규칙

## 목적

AI 코드 생성 시 내재화 확보

## 6단계 프로세스

### 1단계: 기획

- 입력값, 출력값, 주요 로직 명확히 정의
- 테스트 케이스 함께 정의
- 기획 디테일 ↑ = AI 코드 품질 ↑

### 2단계: AI 코드 생성

- 블록(함수/섹션) 단위로 "이 부분이 뭐 하는 건지" 설명 제안
- 호출부 흐름도 함께 안내
- 사용자가 이해 못하면 추가 설명

설명 포함 항목:
- 입력값, 출력값
- 중간 처리 로직
- 조건문/루프 분기 이유
- 호출부 위치와 사용 방식

### 3단계: TDD

- 테스트 코드 작성 → 코드 작성 → 테스트 실행
- 디버거로 변수 값 따라가기
- 조건문/루프에서 "왜 이렇게 분기했나" 확인

### 4단계: 자가 리뷰

코드 작성 완료 시:
- "입력 → 처리 → 출력 흐름 설명해드릴까요?"
- "호출부 3개 따라가볼까요?"

### 5단계: 동료 리뷰

- "이 부분은 왜 이렇게 설계했어요?" 질문에 답할 수 있는 상태로 진입
- 막히면 리뷰어와 함께 파기 → 가장 강력한 학습

### 6단계: 변형 문제 (선택)

- 비슷한 로직을 AI 없이 직접 짜보기
- 막히면 그때 AI나 기존 코드 참고
- 가장 강력한 내재화

## DevOps/인프라 우선순위

### 높음
- Terraform, Helm, 스크립트 (IaC)
- CI/CD 파이프라인
- 흐름 추적 필수

### 낮음
- 복잡한 애플리케이션 로직
- "대강 뭐 하는 건지" 정도면 OK
- 시스템 흐름 파악 > 코드 읽기

## 타임라인

```
AI 코드 생성     : 30분
TDD              : 30분
자가 리뷰        : 5~10분
동료 리뷰        : 20~30분
─────────────────────────
총               : 1.5~2시간
```

## 체크리스트

### 코드 생성 후
- [ ] 각 블록 "뭐 하는 거?" 설명 가능한가
- [ ] 호출부 흐름이 보이는가
- [ ] 불명확한 부분 Claude에게 물었는가

### 테스트 단계
- [ ] 테스트 코드가 시나리오를 커버하는가
- [ ] 디버거로 주요 변수 따라가봤는가
- [ ] 조건문/루프 분기 이유 아는가

### 자가 리뷰 단계
- [ ] 입력 → 처리 → 출력 흐름 그려지는가
- [ ] 호출부 3개 따라갈 수 있는가
- [ ] "왜?" 질문에 답할 수 있는가

### 동료 리뷰 후
- [ ] 설계 결정 설명할 수 있는가
- [ ] 지적받은 부분 이해했는가
- [ ] 다음엔 비슷한 패턴 독자적으로 적용 가능한가
