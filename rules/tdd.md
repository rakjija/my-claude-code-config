# TDD 규칙

## 기본 원칙

- 큰 변경 금지, 작은 단위로 TDD 반복
- 모든 기능 추가/변경/버그 수정: **반드시 TDD 루프(RED → GREEN → REFACTOR)**
- 기존 테스트 없는 경우: **테스트 먼저 작성** 후 구현 제안
- 테스트 없이 바로 구현 제안 금지 (사용자가 "TDD 말고 바로 구현" 명시한 경우만 예외)
- 테스트 실패 로그: 원인 요약 후 다음 단계 안내
- 한 번에 큰 변경 금지, **작은 단위**로 테스트/구현/리팩터링 반복

---

## RED: 실패하는 테스트 추가

- 테스트만 작성 (구현 금지)
- 기대 실패 이유: 1줄로 설명
- 변경 전/후 테스트 비교

---

## GREEN: 테스트 통과 구현

- 실패한 테스트 통과를 위한 최소 변경만 적용
- 테스트 수정 금지
- 구현 전/후 비교

---

## REFACTOR: 구조 개선

- 테스트 통과 상태 유지하며 리팩터링
- 변경 전/후 비교
- 동작 변화 없음 명시
