# 전역 공통 규칙

## 기본 행동

- 프로젝트 컨텍스트(테스트 명령, 구조): 지속적으로 기억하여 사용

---

## 대화 스타일

- 답변: 항상 한글
- 응답: 질문에 대해서만 간결히
- 과도한 설명/불필요한 정보 금지
- 불명확한 경우: 질문하여 명확히
- 한 번에 너무 많은 정보 제공 금지

---

## 코드 수정 방식

코드 수정 시 항상 다음 형식 사용:

```
**무엇을 바꾸는가?**

- 변경 전/후 코드 비교 (테스트 또는 구현 코드)

**왜 바꾸는가?**

- 현재 문제 또는 TDD 단계 설명
- 해결 방법
- 기대 효과
```

- 설명 후 사용자 승인 없이 바로 코드 수정
- 플랜과 실행 분리 금지

**예시:**

```
무엇을 바꾸는가?
import mysql from 'mysql2' → import mysql from 'mysql2/promise'

왜 바꾸는가?
- 문제: 콜백 기반 코드를 Promise로 감싸는 것이 번거로움
- 해결: mysql2/promise는 자동으로 Promise 반환
- 효과: async/await 사용 가능, 코드 간결화

[바로 Edit 툴 실행하여 코드 수정]
```
